Section A — Concept Application

1
How do JavaScript execution, variable scoping (let/const), and DOM selection work together to populate the UI?
▾
JavaScript runs top-to-bottom after the HTML is parsed. Variables declared with let are block-scoped and mutable, while const is block-scoped and read-only — both prevent accidental re-declarations unlike var.

DOM selection (document.querySelector, getElementById) grabs references to HTML elements at runtime. We wait for DOMContentLoaded before querying so the elements actually exist. Once we have a reference, we set textContent, innerHTML, or attributes to inject dynamic profile data without reloading the page.
2
How do conditional logic, truthy/falsy values, and DOM manipulation enable the "Verified Creator" badge?
▾
In JavaScript every value is either truthy or falsy. A number like 0 is falsy; any positive number is truthy.

We write: if (followers >= threshold) — if truthy, we set badge.style.display = 'inline-block', otherwise 'none'. Whenever the follower count input changes we call this check function again, so the badge updates reactively without a page reload. That is DOM manipulation in response to state change.
3
How do arrays, loops, and template literals render and maintain the dynamic social links list?
▾
Each link is stored as an object {title, url} inside an array. When the array changes (add/remove), we loop over it with array.map() and build HTML using template literals:

`<a href="${"$"}{link.url}">${"{link.title}"}</a>`

The joined string is inserted via innerHTML, replacing the entire list in one shot. This ensures the UI always mirrors the array state exactly — no manual DOM patching needed.
4
How do LocalStorage and JSON conversion persist and restore application state across page reloads?
▾
localStorage only stores strings. We use JSON.stringify(array) before saving and JSON.parse() after reading. On page load we check: JSON.parse(localStorage.getItem('links') || '[]').

If we forget to save after a change, the stored version drifts from the in-memory array and the next reload shows stale data. Forgetting JSON.parse gives us a raw string instead of an array, breaking all array methods.
5
How do event handling and event delegation support instant delete without a page refresh?
▾
Instead of attaching a listener to every "Remove" button (which breaks when the list re-renders), we attach one listener to the parent container. This is event delegation:

container.addEventListener('click', e => { if (e.target.matches('.btn-remove')) { ... } })

The click event bubbles up from the button to the container. We read the index from a data-index attribute, splice the array, save to localStorage, and re-render — all synchronously, no reload.
6
How does modular JavaScript (imports/exports) improve maintainability and scalability?
▾
As logic grows, one file becomes a tangled mess. ES Modules let us split concerns: storage.js handles LocalStorage, render.js builds HTML, validate.js checks URLs.

Each module exports only what other modules need and keeps the rest private. The main file imports them. Benefits: easier unit-testing, no global variable collisions, parallel team work on separate files, and tree-shaking in bundlers drops unused code from production builds.
